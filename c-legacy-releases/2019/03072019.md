### Adobe Analytics 中的新增功能 {#aa-features}

| 功能 | 描述 |
| -----------| ---------- |  
| 对区域数据收集的更改 | 为了提高数据收集性能，所有剩余客户已转换为区域数据收集(RDC)，以便收集尽可能接近最终用户。 这可改善站点/应用程序性能，并确保尽快收集数据以优化最终用户体验。<br/>对数字资产中的数据进行地区收集后，这些收集的数据会通过安全连接转发到数据处理中心进行处理，然后提供给 Adobe Experience Cloud 中的产品。RDC 默认用于新实施已长达 10 年以上。[更多](https://marketing.adobe.com/resources/help/zh_CN/reference/regional-data-collection.html) |
| 中国地区的 RDC 更改 | Adobe在中国大陆的区域数据收集服务使中国境内的客户能够直接将数据发送到中国境内的数据收集中心，而不是全球的其他位置。 与将数据发送到中国境外的数据收集中心相比，这提高了页面加载时间和数据准确性。 [更多](https://marketing.adobe.com/resources/help/zh_CN/reference/rdc-china.html) |
| 中国的区域加速 | “中国加速”选项现在在“Analytics报表包设置”中可用。 此选项可改善中国大陆用户的Adobe Analytics 报告用户体验。 每个Adobe Analytics用户都可以启用或禁用此选项。 请勿为中国大陆以外的用户选择此选项，因为这会对性能产生负面影响。 [更多](https://marketing.adobe.com/resources/help/zh_CN/sc/user/report_settings.html) |
| 新的实时流变量 | * **carrier**：移动设备的无线服务提供商。<br/>* **tnt：** Target 中使用的 test-and-target 字段。_actionId_ 和 _actionName_ 从不包含值。<br/>* **tntAction：** Target 中使用的 test-and-target 增强字段。与 tnt 字段不同，_actionId_ 和 _actionName_ 包含值集。 |
| API 2.0：支持 JWT 身份验证 | Adobe Analytics 2.0 API现在支持JWT身份验证。 有关JWT身份验证的详细信息，请访问：[https://github.com/AdobeDocs/analytics-2.0-apis/blob/master/jwt.md](https://github.com/AdobeDocs/analytics-2.0-apis/blob/master/jwt.md) |

### Media Analytics SDK 2.2 适用于：Chromecast、Roku、Apple TV 和 [!DNL Launch] Web Extension {#aa-va}

| 功能 | 描述 |
| -----------| ---------- |
| 用于音频的 Adobe Analytics | 将听众的流音频参与度与其完整的数字行为关联。您可以了解听众对象、收听地点、收听时间和收听频率，并评估用户在常见的分散音频市场中的行为。<br/>这项功能可以让商家评测其听众触及的收听范围、衡量其提供内容的受欢迎程度，并深入分析如何根据用户行为和类众分析模型来保证用户的参与度。使用我们同类中最出色的SDK衡量10秒音频内容和1秒的流广告，以评估体验质量(停顿、错误、开始时间)和内容以及广告KPI。 |
| 维护广告时间 | 在播放器已输入广告中断，并且在特定时间段内未发送呼叫的情况下，SDK逻辑默认为内容播放，即使播放器仍处于广告中断状态，增强广告跟踪功能。 在前滚广告期间，这可能会导致在广告中断完成之前触发内容开始。 在Media 2.2 SDK中，当广告之间存在长时间延迟时，会在广告中断内的广告之间发送缓冲调用。 |
| SDK 名称变更 | 从2.2版本开始，视频心率库(VHL)SDK已更名为Media SDK。 Media 2.2 SDK完全向后兼容VHL 2.X SDK系列。 名称更改不代表功能中断，只是命名约定的更改。 |
