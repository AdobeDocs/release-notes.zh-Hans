**2017 年 8 月 23 日更新**

从8月22日起，Adobe Analytics将开始确定数据收集过程中收到的所有点击的geo_zip。 此项更新提高了数据的准确性，特别是对于移动设备而言。以前，每次访问仅设置一次 geo_zip。现在，geo_zip可以在访客访问过程中正确更改。 geo_zip 字段并不用在报表中，而是显示在数据馈送中。

**2017 年 8 月 21 日更新**

8 月 22 日，Adobe Analytics 将为所有客户启用[地域划分](https://marketing.adobe.com/resources/help/en_US/reference/reports_geosegmentation.html)报表。受此更改影响，预计不久将在 Admin Console 中删除地域划分设置。

发行日期：**2017 年 8 月 17 日**

>[!IMPORTANT]
>
>自 2017 年 9 月发行版起，Ad Hoc Analysis 将终止对低于 1.7.0_76 的 Java 版本的支持。我们的。jar文件已使用安全的256位加密进行签名，低于1.7.0_76的Java版本不支持该加密。 此256位证书使我们能够为您提供增强的安全性。 2017 年 8 月版将继续支持以前的版本。此外，从 2017 年 9 月开始，Ad Hoc Analysis 启动页面将暂时包含一个指向不支持产品版本的链接，以便结合使用低于 2017_1.7.0 的 Java 版本。如果您使用的是 Java 1.7.0_76 或更低版本，请与您的 IT 团队合作，确保尽快升级到更高的 Java 版本。
