## Marketing Cloud 15.9 {#marketing_cloud_interface}

[!DNL  Adobe Marketing Cloud] 界面的新增功能和修复。

发行日期：**2015 年 9 月 10 日**

**文档更新**

解决方案管理员，请参阅[用户和群组](https://marketing.adobe.com/resources/help/en_US/mcloud/user_mgmt_admin.html)，以了解有关将解决方案群组映射到更广泛的 Adobe 群组，以及有关在[企业功能板](http://adminconsole.adobe.html/#)中批量添加用户的重要信息。

**修复**

* 修复了在上传客户属性数据时导致间歇性超时的 Audience Manager API 性能问题。(MAC-26305)
* 修复了导致用户至多向订阅添加 200 个客户属性的问题。(MAC-26188)
* 修复了“客户属性——编辑模式”页面上先前的已知问题，该问题导致在更改显示名称时发出内容识别错误。 （MAC-25589、AN-103834）

**已知问题**

受众库已知问题可能会阻止从Analytics分段进行受众共享，导致受众大小永远保持在0（零）。 如果受众成员数超过50,000，则可能会发生此问题。 作为解决方法，Adobe建议将区段大小减小到50,000个以下受众成员。 (MAC-25788)

请参阅 [Marketing Cloud 产品文档](https://docs.adobe.com/content/help/zh-Hans/core-services/interface/about-core-services/core-services-landing.html)，以获取产品帮助。

## 动态标签管理{#dtm}

**Marketing Cloud 访客 ID 服务工具更新：** 9 月 22 日，DTM 将发布Marketing Cloud 访客 ID 服务工具的更新。最新库版本(1.6.1)将支持改进的客户ID支持、增加对验证状态的支持以及改进的多解决方案集成。

最新版本与通过DTM的Marketing Cloud访客ID服务的当前部署完全兼容。 为了帮助所有用户从此更新中受益，我们将自动将您的帐户迁移到新的Marketing CloudID服务库。

迁移后，您将看到在DTM中为Marketing Cloud访客ID服务工具添加的功能。 我们建议您在发布前在暂存环境中测试此更新。

DTM团队努力为我们的客户增加价值，我们相信您将在Marketing Cloud访客ID服务工具的更新中看到这一点。

**Amazon S3 弃用：** Adobe 不再将 Amazon S3 用作 DTM 库托管选项。

Adobe将不再支持AmazonS3。 您在AmazonS3上的动态标签管理库将一直提供到2015年12月31日，但处于只读状态。 在动态标签管理中所做的任何更改将不再反映在这些文件中。

要在2015年7月31日之后对您的站点部署其他更改，您必须选择我们的其他托管选项之一。 自托管(FTP投放或库下载)始终是建议的最佳实践，但我们的第三方Akamai托管也是世界级的选项。
